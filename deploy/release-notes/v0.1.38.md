## v0.1.38

Image-only release. Polish + corrections on top of v0.1.37's LLM-settings flow.

### What's new

- **Top-right utility bar is theme-only.** The admin LLM-settings gear that briefly lived up there in v0.1.37 moved to the sidebar footer (next to Software Updates, Registry Creds, Change Password, Sign Out). One floating widget for personal preference, every admin/settings action in one row at the bottom of the sidebar.
- **LLM modal: single API-key field per model.** The three-stacked Anthropic / OpenAI / Gemini fields collapsed into one field that swaps as you change models. Pick a Claude model → Anthropic key field. Pick `gpt-5.4` → OpenAI. Pick `gemini-2.5-pro` → Gemini. Pick `ollama/...` → an Ollama base-URL field instead. Picking an unknown-provider model shows a warning.
- **API keys pre-filled with current values.** `GET /api/admin/llm-settings` now returns the actual key strings (admin-only, admin already has root access to the host's `.env`), so the modal can show what's currently configured and let the operator edit in place. Save diffs against the original — only fields you actually changed get written. Picking a new model without touching the key leaves the previously-selected provider's key untouched.

### Operator action

Apply via the Software Updates modal in the UI (one-click). The new top-right utility bar appears on next page reload after the drift-frontend container recreates.

### Notes

- Bundle (install.sh, compose.yml, models.json) is unchanged from v0.1.37. If you're on v0.1.37 you don't need to re-extract the bundle.
- The change to `GET /api/admin/llm-settings`'s response shape is backwards-incompatible at the API level (`configured_keys` boolean dict → real key strings). Only the SPA consumes this endpoint, and it ships in the same release, so no external impact.
