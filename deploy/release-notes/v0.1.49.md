## v0.1.49

Two follow-ups on the v0.1.47 operator-filters feature: **fleet-wide promotion** (any user can share a filter with the whole fleet) and a **Filters sidebar surface** (review / promote / delete without going through chat).

### What's new

**Visibility on `operator_filters`.** New `visibility` column (`'private' | 'fleet'`, default `private`). Lookup semantics:

- Private filters → visible only to the original creator.
- Fleet filters → visible to **every** authenticated operator, regardless of creator.

**New agent tool: `promote_filter(filter_id)`.** Converts a private filter to fleet-wide. Any operator can promote any filter they can see (in practice: their own private filters). Dedup baked in — if an equivalent fleet filter already exists (same canonical pattern + scope), the source row is deleted and the existing fleet row is returned.

**Dedup is now active everywhere a filter is created or promoted.** `remember_filter` checks against (own private ∪ all fleet) and returns the existing match instead of inserting a duplicate. Pattern canonicalization is `strip().casefold()` (Unicode-aware case-fold); scope canonicalization is per-key equality on the four known keys (`device`, `container`, `group`, `signal`). Scope JSON key order doesn't matter — Postgres JSONB compares by value.

**Filters sidebar surface.** New IconButton in the sidebar footer (next to the existing settings icons). Opens a modal showing every filter visible to the operator with:

- Visibility chip (`private` / `fleet`, color-coded)
- Scope chips (one per pinned key — e.g. `device=pi-riffpod-001`, `container=cadvisor`)
- The pattern (monospace) and the operator's `reason`
- Usage signal (`applied N×`, `created <date>`, `last <date>`)
- Promote button (only on your own private filters)
- Delete button (only on filters you created — fleet filters by other operators are read-only to you)

**Ownership / revoke semantics:**

- Anyone can **promote**.
- Only the **original creator** can **forget** (hard-delete) a fleet filter via the agent tool or the sidebar Delete button. Other operators see a tooltip explaining the constraint.
- A future `demote_filter` could let any operator drop a fleet filter back to private — out of scope here.

### REST API

```
GET    /api/filters                      → list visible (own private + all fleet)
POST   /api/filters                      → create private (with dedup)
POST   /api/filters/{id}/promote         → promote to fleet (with dedup)
DELETE /api/filters/{id}                 → hard-delete (only if owned)
```

Cookie auth, same as the rest of the SPA.

### Operator action

Apply via Software Updates modal. Image-only — no install.sh re-run, no edge agent change. Alembic 0013 runs on container startup to add `visibility` (default `'private'` so existing v0.1.47 filters keep working unchanged).

### Cache impact

System prompt section 6 changes (mentions promote_filter + revised ownership rules), and the tools list grows by one entry. Cache invalidates once on first turn after upgrade.
