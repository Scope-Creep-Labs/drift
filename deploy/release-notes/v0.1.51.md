## v0.1.51

Tiny QoL: search across **all** release notes from the Software Updates modal.

### What's new

A search field at the top of the releases section filters releases by case-insensitive substring match across each release's tag, name, and markdown body. Useful when you remember a behavior change but not which release shipped it ("where did we do the `now()` query fix?").

When the search is active:

- The "Previous releases" header switches to `Search results · N of M`.
- The newest release is included in the filter (the "What's new" banner still renders separately above — the search list adds it back when it matches, so you don't have to look in two places).
- Every matching accordion is auto-expanded so the keyword is immediately visible.
- "No release notes mention X" message when there are zero hits.

Clear the field to restore the normal layout.

### Operator action

Apply via Software Updates modal. Frontend-only release — `drift-agent` image is unchanged.
