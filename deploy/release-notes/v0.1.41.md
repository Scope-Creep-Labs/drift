## v0.1.41

Bundle release. Adds per-credential **Validate** button to the admin LLM modal, swaps both validation surfaces (modal + PUT) to LiteLLM's actual completion code path, and externalizes install.sh's validate URLs into `deploy/models.json`.

### What's new

- **Validate button next to every credential field in the admin LLM modal.** Lets the operator confirm a key/model combination works without committing to Save (which triggers a drift-agent restart). Disabled when the field is blank; shows `Testing…` during the probe, `✓ Valid` (green) on success, the provider's error message inline on failure. Same button shape on the Ollama base URL field.
- **Validation runs through LiteLLM, not hand-rolled provider URLs.** Both the new `/api/admin/llm-settings/validate` endpoint and the existing on-Save validation now send a 5-token `litellm.completion(model=…, api_key=…)` request — the exact code path the agent uses at runtime. A green checkmark guarantees the agent's next investigation will authenticate; the previous `/v1/models`-style probe could pass for a key gated to different scopes than what completions need. Cost per click: fractions of a cent on Haiku/Flash, ~$0.005 on Opus.
- **`install.sh` validate URLs come from `models.json`.** Each entry under `providers.<name>.validate_url` is read by `_provider_validate_url()` at install time, with the historical hardcoded URLs as a fallback when the catalog is absent. `package-release.sh --refresh-model-prices` now writes those URLs into the catalog (LiteLLM's JSON doesn't expose them directly, so the small canonical map lives in the refresh script). Net effect: when a provider changes their `/models` endpoint, one map update + one `--refresh-model-prices` run propagates to install.sh without code changes.

### Operator action

```bash
VERSION=v0.1.41
curl -L "https://github.com/Scope-Creep-Labs/drift/releases/download/${VERSION}/drift-deploy-${VERSION#v}.tar.gz" | tar -xz
cd "drift-deploy-${VERSION#v}"
./install.sh
```

For first-time installs of v0.1.41 from any earlier release, the v0.1.39 state-dir refactor applies: the bundle dir is ephemeral (`rm -rf` after install completes); operational state lives at `/var/lib/drift-cp/`.

For operators already on v0.1.40, this is a bundle release because `models.json` and `install.sh` changed. The new providers block is additive — if you skip the install rerun, the running drift-agent's LiteLLM-based validation still works (only the install-time validation reads `models.json`).
