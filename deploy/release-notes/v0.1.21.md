## v0.1.21

Bundle release. Republished from [Scope-Creep-Labs/drift](https://github.com/Scope-Creep-Labs/drift) as the canonical first-install bundle, rebuilt from rewritten git history with redacted credentials. See [v0.1.35](https://github.com/Scope-Creep-Labs/drift/releases/tag/v0.1.35) for the release-source migration context.

## Device tags

- **Free-form tags on devices.** Each device can carry any number of normalized string tags (lowercase, stripped, deduped — `Edge`, ` edge `, and `EDGE` all become `edge`). Existing devices auto-inherit their `group_id` as a tag via the migration.
- **Filter by tag in chat.** `list_devices(tags=["edge", "client-z"])` returns devices matching ALL given tags (match-all). New `tag_device(name, add=[...], remove=[...])` for managing them.
- **Deploy by tag.** New `deploy_revision_to_tags(app, tags=[...])` — same conflict-preflight + offline-skip semantics as `deploy_revision_to_group`. Use case: `deploy reporter to all edge devices for client-z` → `tags=["edge","client-z"]`. Group-based access control still scopes non-admins to their allowed groups.
- **Inline edit in the Devices list.** Each row now shows tag chips next to the group_id and a tag-edit icon (deploy role+) that opens a small dialog for add/remove.
- **REST endpoint**: `PATCH /api/deploy/devices/{name}/tags` with `{set: [...]}` or `{add: [...], remove: [...]}`.

**Bundle release** because of the alembic migration (0010_devices_tags adds the JSONB column + GIN index). On install/upgrade, `alembic upgrade head` runs the migration before drift-agent starts handling requests, then the new tools become available.

## Install

```bash
VERSION=v0.1.21
curl -L "https://github.com/Scope-Creep-Labs/drift/releases/download/${VERSION}/drift-deploy-${VERSION#v}.tar.gz" | tar -xz
cd "drift-deploy-${VERSION#v}"
./install.sh
```

`install.sh` pulls `:latest` images from GHCR, so a fresh install lands directly on the current code regardless of this bundle's version. Subsequent updates apply via the Software Updates modal in the UI (image-only) or by re-running install.sh from a newer bundle tag (bundle updates).
