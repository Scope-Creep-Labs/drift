## v0.1.42

Image-only release. One-line bug fix for the per-session cost display.

### What's fixed

The session token-cost number in the prompt input footer was wrong for any session that used a model other than `claude-opus-4-7`. Cause: `costForUsage(aggregate)` was called without a model argument and silently defaulted to `DEFAULT_MODEL = 'claude-opus-4-7'`, pricing the entire session at Opus rates regardless of the actual model. On a gpt-5.4-mini session this overstated the cost by roughly 22×.

The 30-day sidebar tooltip was unaffected — that path already passes the per-model name through `costForUsageByModel`.

### What changed

`src/components/PromptInput.tsx` now reduces over each turn (live turn included), reads `metadata.engine` (set by the agent on every SSE `start` + metadata event), and prices that turn's tokens against the right per-1M-token rates. Mixed-model sessions price correctly turn-by-turn.

Sanity check on a real number from the bug report: 223,425 tokens that displayed as $1.06 — the actual cost on gpt-5.4-mini's $0.75 input / $4.5 output / $0.075 cache-read rates is roughly $0.046 with the typical ~88% cache-hit ratio. Post-fix, the display shows that.

### Operator action

Apply via the Software Updates modal in the UI (one-click). Image-only — no install.sh re-run.
