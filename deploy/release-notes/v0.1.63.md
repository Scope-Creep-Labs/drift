## v0.1.63

Bump edge-agent `AGENT_VERSION` to `0.14.0` (from `0.13.3`).

### Why

`AGENT_VERSION` is the human-readable string each device reports on check-in (stored in `devices.agent_version`, rendered in the sidebar). It hadn't been touched since well before v0.1.59 even though `drift-deploy-agent.sh` itself picked up substantial behavior changes:

- v0.1.59 — `pending_tunnels[]` handling + `tunnel_bridge_target_sha` self-update for the new tunnel feature.
- v0.1.60 — first-provision path for `tunnel-bridge.py` so existing devices auto-fetch the file without an `install.sh` rerun, plus a WARN log when the bridge file is missing but tunnels are pending.

Bumped MINOR (`0.13.3` → `0.14.0`) since the agent gained new responsibilities (tunnel forwarding alongside terminal relay), not just bug fixes.

### Operator action

Apply via Software Updates modal. Each device's `drift-deploy-agent.sh` self-updates on its next check-in (≤30s); the sidebar will then show `0.14.0` for every online device. No per-device install.sh needed.

### Migration / breaking changes

None. Check-in payload + response schema unchanged.

### Cache impact

No system prompt or tool registry changes.
