## v0.1.37

**Bundle release.** Picks up changes to `install.sh`, `docker-compose.yml`, and a new `deploy/models.json` catalog file. Operators on v0.1.36 or earlier need to download this tarball and re-run `install.sh` (existing `.env` is preserved; just hit Enter through prompts to keep current values).

### What's new

- **LLM model picker in `install.sh`.** Replaces the free-text model prompt with a tiered menu (Premium / Fast / Local) showing pricing and provider per model. Operator picks by number or types an ID for anything not in the curated catalog. Provider is auto-detected so only the matching API key gets prompted (Ollama tier prompts for `OLLAMA_API_BASE` instead). Catalog lives in `deploy/models.json` and is regenerated from LiteLLM via `./deploy/package-release.sh --refresh-model-prices`.

- **Admin LLM-settings modal in the UI.** Top-right gear icon (admin role only) opens a modal that lists models from LiteLLM's full registry with inline per-token pricing, plus fields for the Anthropic / OpenAI / Gemini API keys. "Recommended" favorites are grouped at the top. Save writes the changed fields to the installer's `.env` (atomic write, preserves comments and unrelated vars) and triggers a detached `drift-agent` recreate. The page auto-reloads ~8s after save so the modal's "what you're running now" view matches reality.

- **Runtime pricing fed by LiteLLM.** New `GET /api/models/pricing` endpoint returns per-1M-token costs from `litellm.model_cost`, filtered to chat models with non-zero pricing. Frontend's `pricing.ts` consumes it (with the existing hardcoded table as fallback for first paint and graceful degradation). Token-cost display in the sidebar now works correctly for any LiteLLM-supported model without per-vendor maintenance.

- **Theme toggle relocated.** Moved out of the sidebar footer and into a floating top-right utility bar (translucent backdrop, blur, fixed position). Holds the theme cycle on its own row and a gear icon (admin only) for the LLM settings modal.

- **`OLLAMA_API_BASE` setting.** First-class config knob for local-model deployments. Installer prompts for it when an Ollama model is picked; the admin modal exposes it for runtime changes.

### Operator action

```bash
VERSION=v0.1.37
curl -L "https://github.com/Scope-Creep-Labs/drift/releases/download/${VERSION}/drift-deploy-${VERSION#v}.tar.gz" | tar -xz
cd "drift-deploy-${VERSION#v}"
./install.sh
```

`.env` survives across re-installs (sidecar at `/var/lib/drift-cp/.env`). Hit Enter through prompts to keep current values; pick a model from the new menu or type your current one to skip.

### Security notes

- `.env` mode bumps from `640` to `660` (root:docker) so drift-agent can write to it from the admin LLM-settings endpoint. The docker group is already root-equivalent via `/var/run/docker.sock`, so this isn't a new trust boundary.
- API keys are never returned from `GET /api/admin/llm-settings`; only a boolean flag per provider indicating whether the key is set. Operators rotate keys by typing the new one into the field and saving.
- The modal is admin-role-only at both the route handler and the UI surface.

### Migration / breaking changes

None. Existing `.env` files continue to work; the new `OLLAMA_API_BASE` setting defaults to empty (no-op unless an Ollama model is configured). The runtime pricing endpoint falls back to the previous hardcoded table on failure.
