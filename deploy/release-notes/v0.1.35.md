## v0.1.35

Image-only release. **Release source moved**: this is the first release published from `Scope-Creep-Labs/drift` (was `kidproquo/drift-public`). The Software Updates modal in v0.1.35+ polls the new source.

### What's new

- **Source repo + release channel consolidated.** `kidproquo/drift-public` is being deprecated; the canonical source and release home is now [Scope-Creep-Labs/drift](https://github.com/Scope-Creep-Labs/drift). The repo is fully public.
- **`RELEASES_REPO` env var.** The admin updates poller's GitHub-releases source is now overridable via `DRIFT_RELEASES_REPO`. Default is `Scope-Creep-Labs/drift`. Useful for forks or staging release channels.
- Docs (`DEPLOY.md`, `deploy/UPDATES.md`, `deploy/README.md`, `CONTRIBUTING.md`, `README.md`) updated to reference the new source.

### Operator action (one-time)

The currently-running drift-agent (v0.1.34 or earlier) is still polling the old `kidproquo/drift-public` source, which won't see this release. To roll forward:

```bash
docker compose pull drift-agent && docker compose up -d drift-agent
```

Once v0.1.35 is running, the Software Updates modal polls `Scope-Creep-Labs/drift` automatically. After that point, future image-only releases land in the modal as usual.

### GHCR images

Unchanged: still at `ghcr.io/kidproquo/drift-agent` and `ghcr.io/kidproquo/drift-frontend`. Only the GitHub repo and release channel moved. GHCR packages are owned independently of the GitHub repo and don't need to be retagged.

### Earlier release notes

For v0.1.22 through v0.1.34 (which were published only on the old source), see [kidproquo/drift-public/releases](https://github.com/kidproquo/drift-public/releases) before that repo is removed. The summary: fingerprint TOFU + case-insensitive device names (v0.1.33), reinstall-from-existing-env shortcut (v0.1.34).
