## v0.1.33

Image-only release. Cross-host paste protection for device commissioning.

### What's new

- **Host fingerprint TOFU.** Edge agents now send a sha256 of `/etc/machine-id` (with fallback to `/var/lib/dbus/machine-id`, then `/sys/class/dmi/id/product_uuid`) on every check-in. The CP records it on the first check-in after commissioning, then validates on every subsequent one. Mismatches return HTTP 409 with a clear remediation message. Stops accidental cross-host paste of the commissioning curl from silently letting two machines share one device identity.
- **Case-insensitive, whitespace-trimmed device names.** `commission_device` rejects duplicates regardless of casing or surrounding whitespace. Names are normalized (strip + lower) at every write boundary. Deleting a device frees its name for reuse (lenient tombstones).
- **`commission_device` guidance** rewritten to reflect the real bootstrap-token lifetime. The token is the device's long-lived bearer credential, not a single-use secret. Save the rendered install_cmd to a password manager so you can reinstall on the same host later; fingerprint TOFU prevents reuse on a different host.
- **DEPLOY.md** updated with the cross-host paste scenarios.

### Operator action

After applying this release, re-run `install.sh` on each existing edge device. The new install adds bind-mounts (`/etc/machine-id` etc.) that the agent needs to compute its fingerprint. Until each device is re-installed, its check-ins send an empty fingerprint and the CP doesn't enforce TOFU. Existing behavior preserved (backwards-compatible); the protection just lights up per-device as you re-install.

### Database migration

Alembic `0011` runs on `drift-agent` startup:

- Adds `devices.host_fingerprint VARCHAR(64)` nullable column.
- Normalizes existing device names in place (`UPDATE devices SET name = LOWER(TRIM(name))`).
- Replaces the case-sensitive `devices_name_key` UNIQUE constraint with a partial unique index on `LOWER(name) WHERE status != 'removed'`.

If your fleet contains case-collision device names (e.g. `Pi-001` and `pi-001`), the migration fails fast with a clear error listing the collisions. Resolve in the DB (rename or delete duplicates) and re-run.

### Edge agent

Bumped to `AGENT_VERSION=0.12.0`. Self-updates from the CP on the next poll cycle (~30s per device); the new install.sh bind-mounts the fingerprint sources.
