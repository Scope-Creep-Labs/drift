## v0.1.54

Fix: terminal modal chrome (title, status badge, X button, bottom helper text) was invisible in light mode.

### Why

The Dialog Paper for `TerminalModal` is hardcoded to `bgcolor: '#0b0b0d'` — terminals are always dark regardless of the app theme. But every default-themed child in the title bar (Typography, IconButton's CloseIcon, the `color="default"` Chip, and the bottom helper text using `color="text.disabled"`) inherits the app's theme palette, which is **dark text** in light mode. Dark text on a near-black Paper = invisible.

In dark mode the bug went unnoticed because dark text on near-black happens to share `#0b0b0d`-ish values with the theme's `text.primary`, just barely visible.

### What changed

`TerminalModal.tsx`:

- DialogTitle now sets `color: 'common.white'` so Typography + Icons in the title bar inherit white.
- The status Chip with `color="default"` (phases `waiting` / `creating`) gets an explicit `sx` overriding `color` + `borderColor` to light-mode-safe values. Success / error variants already pop on dark and are left alone.
- The bottom "Log in as drift…" helper text drops `color="text.disabled"` in favor of `rgba(255, 255, 255, 0.5)` — dim enough to recede, light enough to read.

### Operator action

Apply via Software Updates modal. Frontend-only release — `drift-agent` image unchanged.
