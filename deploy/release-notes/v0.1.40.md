## v0.1.40

Image-only release. Two upgrades to the admin LLM-settings modal.

### What's new

- **Reasoning effort is editable from the modal.** New `Reasoning effort` select between the model picker and the API-key field — `low / medium / high / xhigh / max`. Passed to thinking-capable models (Opus, o1/o3, Gemini 2.5 Pro/Flash); ignored by models that don't support thinking. Backend already accepted `effort` in PUT and exposed it from GET; the modal just hadn't surfaced it.
- **API keys are validated against the provider before being written.** Save now triggers a parallel probe of each credential the operator changed:
  - Anthropic → `GET api.anthropic.com/v1/models` with `x-api-key`
  - OpenAI → `GET api.openai.com/v1/models` with `Authorization: Bearer`
  - Gemini → `GET generativelanguage.googleapis.com/v1beta/models?key=…`
  - Ollama → `GET <base>/api/tags` (validates reachability AND that the host is actually serving Ollama)

  Failure surfaces as a `400` with a per-field message; the modal routes each message back to the matching input as an inline error. No bad creds get written to `.env`. Probes run concurrently via `asyncio.gather`, so the worst-case wait is one provider's latency (~1–3s on a healthy network), not the sum.

### Operator action

Apply via the Software Updates modal in the UI (one-click). Image-only — no install.sh re-run needed; `/var/lib/drift-cp/` is unchanged.
