## v0.1.34

Image-only release. Documentation only — no behavior changes.

### What's new

- **Reinstall shortcut documented.** Re-running `install.sh` on an already-commissioned device no longer requires re-supplying `DEVICE_NAME` / `BOOTSTRAP_TOKEN` / `CP_URL` / `GROUP_ID`. Source the existing `/etc/drift-deploy/env` and pipe install.sh:

  ```bash
  sudo bash -c 'set -a; . /etc/drift-deploy/env; set +a; \
      unset CURL_CA_BUNDLE SSL_CERT_FILE; \
      curl -fsSL "$CP_URL/agent/install.sh" | bash'
  ```

  Useful for image-baseline updates (like v0.1.33's `/etc/machine-id` bind-mount) where you want to rerun on every device but don't want to dig original credentials out of a password manager.

### Where it's documented

- `edge-agent/install.sh` header comment block.
- `DEPLOY.md` Scenario 5 (commission a new device).
- `commission_device` chat tool's `guidance` field, so the agent surfaces it whenever a new device is commissioned.

### Why the unset

`/etc/drift-deploy/env` sets `CURL_CA_BUNDLE` and `SSL_CERT_FILE` to container-only paths (`/host/etc/ssl/host-ca-bundle.crt`) for use inside the drift-deploy-agent container. When sourced on the host, those values break curl. Unsetting them lets curl fall back to the host's system CA bundle; install.sh re-detects the host's bundle on its own and re-bakes the correct container path on the next run.
