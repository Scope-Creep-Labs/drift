## v0.1.67

**SPA modal for the Telegram `/link` flow** + QR code in the response so users can bind a chat by scanning their phone instead of typing a code.

### What's new

**Backend:**

- New dep `segno>=1.6` (added to `pyproject.toml` and the Dockerfile install list). Pure-Python QR generator, no native build, tiny.
- `app/telegram/api.py`: new `_qr_data_uri(data)` and `deep_link_and_qr(code) -> (link, qr_data_uri)`. The QR is a PNG `data:` URI ready to drop into `<img src=…>` — no extra HTTP fetch.
- `POST /api/telegram/link/code` now returns `qr_data_uri` alongside `code`, `deep_link`, `expires_at`, `expires_min`. Both `deep_link` and `qr_data_uri` are nullable: graceful degradation if `TELEGRAM_BOT_TOKEN` is set but the bot's `getMe` failed, or if `segno` isn't installed (older runtime layer).

**Frontend:**

- `src/components/TelegramLinkModal.tsx` — modal with three sections:
    - **Generate code**: button → API call → displays the 6-char code in big monospace, the QR alongside (scannable from a phone), an "Open in Telegram" button using the deep link, and a copyable `/link CODE` fallback for manual entry.
    - **Your linked chats**: polled every 4s so a newly-redeemed code shows up without a manual refresh. Each row has a delete icon → unlink.
    - Friendly error if `TELEGRAM_BOT_TOKEN` is unset on the CP (the API returns 503; the modal surfaces a clear "ask your admin to set TELEGRAM_BOT_TOKEN" line rather than a raw HTTP error).
- `src/state/telegramUiStore.ts` — single-instance open/close store; pattern mirrors `terminalUiStore` and `tunnelUiStore`.
- Trigger lives in the sidebar bottom-row icon cluster, between **Noise filters** and **Change password** — available to every logged-in user, hidden in DEMO_MODE (same posture as the other per-user surfaces).
- Mounted at the App root in `Shell.tsx` so any future surface (a chat-action card, a setup-complete banner, etc.) can open it through the store.

Backend bot loop unchanged from v0.1.66 — the long-poll → bot → agent → reply path remains intact; this release just gives users a self-serve onboarding UI.

### Operator action

Apply via Software Updates modal. Both `drift-agent` and `drift-frontend` images rebuild (backend Dockerfile + new SPA component touch both).

Users can then:

1. Click the Telegram icon in the sidebar bottom row
2. **Generate code** → scan the QR on their phone (or tap "Open in Telegram")
3. The bot confirms with ✅ Linked
4. Their newly-linked chat appears in "Your linked chats" within 4s

No env changes if you're already on v0.1.66 with `TELEGRAM_BOT_TOKEN` set.

None.
