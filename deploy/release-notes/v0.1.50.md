## v0.1.50

Per-user **mute** on operator filters — opt out of any filter (your own private, your own fleet, or someone else's fleet) without removing the row.

### Why

v0.1.49 left a gap: a non-creator on the receiving end of a fleet filter had no way to opt out. Only the original creator could delete; everyone else was stuck with the rule. Mute closes that gap, and the same mechanism turns out to be useful for the other two filter classes too:

- **Your own private** — mute keeps the row around but skips applying it for the current debugging session. Saves the delete-and-recreate dance for short-lived opt-outs.
- **Your own fleet** — you promoted it for the team but want raw data in your own investigations. Mute opts you out without affecting others.
- **Someone else's fleet** — the only way out, since you can't delete what you didn't create.

### What's new

**New table `operator_filter_mutes`** with composite PK `(user_id, filter_id)`, both columns ON DELETE CASCADE. Mute is idempotent (insert is a no-op if a row already exists), unmute is just `DELETE`.

**Two new agent tools:**

- `mute_filter(filter_id)` — per-user opt-out. Works on any visible filter; idempotent.
- `unmute_filter(filter_id)` — reverses it; idempotent.

**`list_relevant_filters` excludes muted rows** for the calling operator. The sidebar `GET /api/filters` keeps them visible (with `muted_by_me: true`) so the operator can flip the toggle back.

**REST API additions:**

```
POST   /api/filters/{id}/mute     → mute for the calling user
DELETE /api/filters/{id}/mute     → unmute for the calling user
```

**Sidebar UI:**

- Mute toggle (speaker icon) on every row, regardless of ownership or visibility.
- Muted rows render with reduced opacity + a yellow "muted" chip.
- Hover the mute icon: "Mute: stop applying this filter in YOUR investigations (others unaffected)" / "Unmute: apply this filter again".
- The "you can't delete someone else's fleet filter" tooltip now points to mute as the alternative.

### Semantics summary

| Filter type | Promote | Delete | Mute (per-user) |
|---|---|---|---|
| Private (yours) | ✓ | ✓ | ✓ |
| Fleet (yours) | — already promoted | ✓ | ✓ |
| Fleet (someone else's) | — already promoted | ✗ (creator only) | ✓ |

### Operator action

Apply via Software Updates modal. Image-only — no install.sh re-run, no edge agent change. Alembic 0014 runs on container startup.

### Cache impact

Tools list grows by two entries; system prompt section 6 expanded for mute/unmute behavior. One cache miss on first turn after upgrade, then steady-state caching resumes.
