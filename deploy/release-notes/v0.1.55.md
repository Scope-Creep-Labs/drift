## v0.1.55

Fix-on-fix: terminal modal close button still invisible in light theme after v0.1.54.

### Why

v0.1.54 set `color: 'common.white'` on the DialogTitle expecting all descendants to inherit, but MUI's `IconButton` defaults to `color="default"` which applies `palette.action.active` (`rgba(0,0,0,0.54)` in light mode) as an **explicit** CSS color on the element. Explicit beats inherited — the close icon ignored the parent's white and stayed dark gray on the dark Paper.

### What changed

Added `color="inherit"` to the close `IconButton`. With explicit-inherit it picks up the DialogTitle's `common.white`, and the SvgIcon's natural `fill: currentColor` follows.

### Operator action

Apply via Software Updates modal. Frontend-only — `drift-agent` image unchanged.
